Описание
Modal
Значение: 0. Команду нельзя вызвать, пока активна другая команда.
Transparent
Значение: 1. Команду можно вызвать, пока активна другая команда. Если этот флаг установлен, функцию NRX ncedSSSetFirst() использовать нельзя.
UsePickSet
Значение: 2. При извлечении набора объектов, выбранных перед запуском команды (набор pickfirst), этот набор в nanoCAD очищается. Команда может извлечь набор pickfirst с помощью функций NRX ncedSSSetFirst() или ncedSSGet("I."). Команда может задать набор pickfirst с помощью ncedSSSetFirst(), но он остается заданным только до завершения команды. Команда не может извлекать или задавать ручки.
Redraw
Значение: 4. При извлечении набора объектов, выбранных перед запуском команды (набор pickfirst), или набора ручек они не очищаются в nanoCAD. Команда может извлечь эти наборы с помощью функции NRX ncedSSSetFirst(). Команда может извлечь набор pickfirst с помощью ncedSSGet("I."). Команда может задавать как набор pickfirst, так и набор ручек с помощью функции NRX ncedSSSetFirst(). Объекты в этих наборах перерисовываются с соответствующими маркерами ручек и выделением после завершения команды.
NoPerspective
Значение: 8. Вызов команды не допускается, пока в текущем видовом экране установлен вид в перспективе.
NoMultiple
Значение: 16. Команда с таким флагом не допускает своё многократное выполнение после команды MULTIPLE. В текущей версии nanoCAD использование этого флага не реализовано.
NoTileMode
Значение: 32 (0x20). Не допускается вызов команды, если TILEMODE == 1. Команду с таким флагом можно будет вызвать только в пространстве модели.
NoPaperSpace
Значение: 64 (0x40). Не допускается вызов команды, когда активно пространство листа.
PlotOnly
Значение: 128. Команда с таким флагом допускает выполнение только в процессе вывода на печать.
NoOem
Значение: 256. Команда с таким флагом не будет доступна для OEM-продуктов.
Undefined
Значение: 512 (0x200). Команду можно вызвать только с использованием синтаксиса cmdGroupName.cmdGlobalName.
InProgress
Значение: 1024 (0x400). Команда с таким флагом может выполняться в процессе выполнения другой команды.
Defun
Значение: 2048 (0x800). Команда может быть вызвана как функция LISP и, следовательно, может использовать функцию NRX ncedGetArgs() для получения аргументов из LISP и может использовать функцию NRX ncedRetXxx() для возврата значений в LISP. Этот флаг может быть установлен только в Visual LISP.
NoNewStack
Значение: 65536. Команда с таким флагом не будет помещена в новый стек команд.
NoInternalLock
Значение: 131072 (0x20000). На текущую команду с таким флагом не будет дополнительно накладываться внутренняя блокировка.
DocReadLock
Значение: 524288 (0x80000). Если этот флаг установлен, то при вызове команды выполняется блокировка документа для чтения (read lock). Если флаг не установлен, осуществляется блокировка документа для записи (write lock).
DocExclusiveLock
Значение: 1048576 (0x100000). Если этот флаг установлен, то при вызове команды выполняется исключительная блокировка документа (exclusive lock). Если флаг не установлен, осуществляется совместная блокировка документа (shared lock).
Session
Значение: 2097152 (0x200000). Команда будет выполняться в контексте приложения, а не в контексте текущего документа, с различными возможностями и ограничениями, которые это влечет. Ее следует использовать в меру.
Interruptible
Значение: 4194304 (0x400000). Команда может быть прервана при запросе ввода данных пользователем во время вызова любого из методов acedGetxxx.
NoHistory
Значение: 8388608 (0x800000). Команда не будет использоваться механизмом повторения последней команды nanoCAD.
NoUndoMarker
Значение: 16777216 (0x1000000). Команда и ее маркеры отмены не будут добавлены в файл отмены. Этот флаг предусмотрен для команд, которые не изменяют базу данных и, следовательно, не нуждаются в отображении в файле отмены.
NoBlockEditor
Значение: 33554432 (0x2000000). Команда с таким флагом не доступна в редакторе блоков.
NoActionRecording
Значение: 67108864 (0x4000000). Этот флаг вынуждает ActionRecorder игнорировать связанную команду.
ActionMacro
Значение: 134217728 (0x8000000). Этот флаг определяет команды, которые создаются в результате записи действий с помощью ActionRecorder.
Описание

Данное перечисление определяет значения, которые могут быть переданы как аргумент commandFlags функции NRX NcEdCommandStack.addCommand(). В .NET API nanoCAD эти значения могут быть переданы как аргумент flags в метод Teigha.Runtime.CommandMethodAttribute(). 

Функция addCommand() создает объект NcEdCommand, передает ему копии глобального и локального имён команды, связанные флаги команд и функцию, которая должна быть выполнена командой; и затем добавляет созданный объект в группу команд. Если имени группы команды не существует, то создается новая группа команд и добавляется в CommandStack перед добавлением нового объекта NcEdCommand. В противном случае выполняется поиск существующей группы команд для поиска объекта NcEdCommand с тем же глобальным именем, что и доступное глобальное имя команды, или с тем же локальным именем, что и доступное локальное имя команды.
