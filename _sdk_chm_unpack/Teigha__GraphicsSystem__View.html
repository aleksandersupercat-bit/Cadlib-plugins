Класс View является абстрактным базовым классом для классов, реализующих объекты видового экрана Viewport в окнах графического пользовательского интерфейса nanoCAD. Каждый объект Viewport включает определяющие вид параметры камеры и коллекцию отображаемых объектов.

Описание

Объект View представляет некоторую часть окна графического пользовательского интерфейса и включает определяющие вид параметры камеры и коллекцию отображаемых объектов. Другими словами, объекты View можно использовать для представления видового экрана nanoCAD. 

Объект View добавляется к устройству Device. В каждый конкретный момент времени объект View может принадлежать только одному объекту Device. Виду следует указать, что именно он должен отображать. Клиент добавляет и удаляет пары (отображаемый объект Drawable, модель Model), соответствующие отображаемому фрагменту модели. Каждому виду задается режим рендеринга RenderMode, который соответствует алгоритму отрисовки геометрии в этом виде. 

Объект View имеет два состояния: интерактивное и неинтерактивное. По умолчанию используется неинтерактивное состояние, в котором все методы выполняются немедленно при вызове с возвратом только после завершения. Все возвращаемые значения могут рассматриваться как надежные, и вызываться могут все методы. 

Вызов метода BeginInteractivity() переводит объект View в интерактивное состояние с указанием минимальной частоты кадров, которую вид будет пытаться поддерживать. Для этого он ставит в очередь все изменения вида и вызовы обновления, обрабатывая их асинхронно. Здесь важно отметить, что минимально приемлемая частота кадров часто отличается от темпа поступления пользовательских данных (например, от частоты получения координат курсора мыши). Вследствие асинхронной обработки поставленных в очередь изменений вида небезопасно вызывать методы, которые запрашивают или изменяют состояние вида (например, Add() или Erase()). К надежным можно отнести только вызовы следующих свойств и методов: SetView(), EnableFrontClip, FrontClip, EnableBackClip, BackClip, Mode, Dolly(), Roll(), Orbit(), Zoom(), Pan(), Update(), Flush() и EndInteractivity(). При возврате из метода EndInteractivity() вид переводится из интерактивного в неинтерактивное состояние. 

Объекты View поддерживают переднюю и заднюю плоскости отсечения. Эти плоскости ориентированы так, чтобы их нормали совпадали с вектором взгляда, и их ориентация сохраняется при изменении вектора взгляда. Передняя плоскость отсечения, если включена, подавляет отображение любой геометрии, которая располагается между камерой и передней плоскостью отсечения. Соответственно, задняя плоскость отсечения, если включена, подавляет отображение любой геометрии, которая удалена от камеры в направлении вектора взгляда дальше задней плоскости. 

Класс View следует использовать для доступа к полному описанию вида в режиме перспективы. Он поддерживает весь необходимый функционал для определения вида, плоскостей отсечения и матриц преобразования вида.
