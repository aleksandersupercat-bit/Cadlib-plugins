Метод вычисляет точки пересечения между текущим геометрическим объектом и указанным в параметре entityPointer, включая пересечения с ребрами ограничивающего прямоугольника. Найденные точки добавляются в коллекцию points в мировых координатах (WCS), а тип пересечения задаётся через параметр intersectType. Параметры thisGraphicSystemMarker и otherGraphicSystemMarker позволяют ограничить поиск пересечений конкретными подобъектами, но могут быть проигнорированы, если установлены в 0.

Описание

Метод IntersectWith вычисляет точки пересечения между текущим геометрическим объектом и другим объектом, указанным через параметр entityPointer, включая пересечение с ребрами ограничивающего прямоугольника (bounding box) текущего объекта. Тип пересечения задается параметром intersectType, который определяет, требуется ли продление одного или обоих объектов для поиска точек пересечения. 

Все найденные точки пересечения добавляются в коллекцию points в мировой системе координат (WCS). Для локализации поиска пересечений между конкретными подобъектами используются параметры thisGraphicSystemMarker и otherGraphicSystemMarker. Эти параметры содержат графические маркеры, указывающие на подобъекты текущего и целевого объектов соответственно. Если значение какого-либо из них меньше или равно 0 — соответствующий параметр игнорируется. 

При реализации метода в производных классах рекомендуется использовать встроенные графические библиотеки и их функционал (например, методы IsOn() или IntersectWith() из классов Curve3d и CircularArc3d) для поиска пересечений на уровне примитивов. Если тип объекта entityPointer не поддерживается напрямую, можно вызвать его собственный метод IntersectWith(), передав ему текущий объект в качестве аргумента.
