Как использовать технологию EntityJig
Послать отзыв

Технология позволяет создавать пользовательский примитив непосредственно на экране путем указания начальной точки и перетаскивания курсора мыши (далее технология Jig).

Описание

Технология позволяет создавать пользовательский примитив непосредственно на экране путем указания начальной точки и перетаскивания курсора мыши (далее технология Jig). При этом во время перетаскивания курсора мыши на экране будет отображаться предпросмотр создаваемого примитива.

Пример

 

Шаги, которые нужно выполнить, чтобы реализовать команду построения примитива с помощью технологии Jig:

  1. Создать класс-наследник класса EntityJig:
    • Реализовать в этом классе конструктор, принимающий в качестве одного из параметров переменную типа создаваемого примитива (Line, Circle и т.п.). Тип создаваемого примитива должен быть наследником класса Entity.
    • Реализовать в классе метод Sampler(). В методе необходимо вызвать один из методов класса JigPrompts. Метод отслеживает перемещение курсора мыши, получая и запоминая данные о его положении.
    • Реализовать в классе метод Update(). Метод преобразовывает полученные методом Sampler() данные в формат, необходимый для построения примитива на экране.
  2. Создать команду построения примитива:
    • В методе команды сделать запрос данных, необходимых для первоначальной инициализации примитива.
    • Запомнить полученные данные.
    • Запустить транзакцию с базой данных чертежа. Далее все действия происходят внутри транзакции.
    • Создать экземпляр класса создаваемого примитива.
    • Создать экземпляр класса из п.1, передав в него ссылку на экземпляр создаваемого примитива и данные, полученные запросом данных для первоначальной инициализации примитива.
    • Вызвать метод Editor.Drag(), передать в него экземпляр класа из предыдущего пункта.
    • Проверить, что вернул метод Editor.Drag(). Если он вернул PromptStatus.OK, добавить примитив в базу данных чертежа и завершить транзакцию.

 

В результате работы примера на пространстве чертежа создается полукруг из пользовательского примитива Arc.

public SemiсircleEntJig(Arc semicircle, Point3d point) : base(semicircle) { this.basePoint = point; this.newPoint = point; } protected override SamplerStatus Sampler(JigPrompts prompts) { // Запрос второй точки для расчета радиуса полукруга JigPromptPointOptions options = new JigPromptPointOptions("Радиус полукруга:"); PromptPointResult promptResult = prompts.AcquirePoint(options); // Проверка полученных данных if (promptResult.Status == PromptStatus.OK) { if (this.basePoint == promptResult.Value) { return SamplerStatus.NoChange; } else { this.newPoint = promptResult.Value; return SamplerStatus.OK; } } return SamplerStatus.Cancel; } protected override bool Update() { // Инициализация дуги Arc semicircle=Entity as Arc; if (semicircle == null) return false; semicircle.Center = this.basePoint; semicircle.StartAngle = 0;semicircle.EndAngle = Math.PI; // Расчет радиуса полукруга по двум точкам Line radius = new Line(this.basePoint, this.newPoint); semicircle.Radius = radius.Length; return true; } } public class Commands { [CommandMethod("SemicircleEntityJig")] public void SemicircleJigCommand() { Document doc = Application.DocumentManager.MdiActiveDocument; Editor ed = doc.Editor; Database db = doc.Database; // Запрос центра полукруга PromptPointOptions promptOptions = new PromptPointOptions("Центр полукруга:"); PromptPointResult promptResult = ed.GetPoint(promptOptions); // Проверка полученных данных if (promptResult.Status != PromptStatus.OK) return; Point3d point = promptResult.Value; using (Transaction transaction = db.TransactionManager.StartTransaction()) { // Получение доступа к активному пространству (пространство модели или лист) BlockTableRecord currentSpace = transaction.GetObject(db.CurrentSpaceId, OpenMode.ForWrite) as BlockTableRecord; // Определение экземпляров примитива и класса-наследника EntityJig Arc semicircle = new Arc(); SemiсircleEntJig semicircleEntJig = new SemiсircleEntJig(semicircle, point); // Вызов метода, позволяющего отрисовывать геометрические объекты путем перетаскивания курсора мыши promptResult = (PromptPointResult)ed.Drag(semicircleEntJig); if (promptResult.Status == PromptStatus.OK) { // Добавление дуги в чертеж currentSpace.AppendEntity(semicircle); transaction.AddNewlyCreatedDBObject(semicircle, true); transaction.Commit(); } } } }

Алгоритм работы приведенного примера:

  1. Запускается работа команды SemicircleEntityJig, выполняется метод SemicircleJigCommand() класса Commands.
  2. Запрос центра полукруга, проверка полученных данных.

  1. Если введенные данные верны, создается примитив Arc и экземпляр класса SemicircleEntJig, позволяющий построить полукруг с помощью технологии Jig. При создании экземпляра этого класса в его конструктор передается ссылка на созданный ранее примитив Arc и координаты центра будущего полукруга.
  2. Вызов метода Drag() класса Editor, в который передается экземпляр класса SemicircleEntJig. Метод обращается к экземпляру класса SemicircleEntJig при каждом передвижении курсора мыши на экране.
  3. Запрос данных для расчета радиуса полукруга методом Sampler() класса SemicircleEntJig.
  4. Метод Sampler() получает точку расположения курсора мыши на экране и запускается метод Update() класса SemicircleEntJig, в котором из полученной точки высчитывается радиус полукруга.
  5. После выполнения метода Update() на экране появляется полукруг. При перемещении курсора мыши будет происходить пересчет радиуса полукруга и его изменение на экране.

 

  1. После того, как радиус полукруга был выбран нажатием кнопки мыши на экране или путем ввода с клавиатуры, работа метода Drag() класса Editor завершается, созданный примитив добавляется в базу данных чертежа. Работа команды SemicircleEntityJig завершается.
