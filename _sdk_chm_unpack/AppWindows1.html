Как создавать WPF/WinForms - окна
Послать отзыв

Статья о том, как создавать WPF/WinForms - окна, связанные с редактором dwg-документа в nanoCAD.

Описание

При помощи API nanoCAD можно создавать дополнительные модальные и немодальные окна, а также палетки, которые являются разновидностью немодальных окон. 

Для создания WPF/WinForms - окон сначала необходимо создать UserControl. Для вызова этих окон необходимо использовать методы класса HostMgd.ApplicationServices.Application. Эти методы обеспечат взаимодействие окон с редактором nanoCAD. При запросе данных с чертежа с использованием таких методов, окна будут автоматически пропадать, пока данные с чертежа не будут получены и после этого автоматически снова появляться: 

 

  • Методы для вызова окон, сделанных с помощью WinForms:
    • ShowModalDialog() - для модального окна;
    • ShowModelessDialog() - для немодального окна.

 

  • Методы для вызова окон, сделанных с помощью WPF:
    • ShowModalWindow() - для модального окна;
    • ShowModelessWindow() - для немодального окна.

 

Для создания палетки необходимо также создать UserControl на WinForms или WPF. Но, чтобы добавить палетку к интерфейсу, необходимо сначала создать набор палеток HostMgd.Windows.PaletteSet(), даже если планируется создание только одной палетки. После этого в набор палеток добавляются дочерние палетки. Для каждой дочерней палетки необходимо создавать UserControl, если они отличаются. Палетки, в отличие от окон, не вызываются методами класса HostMgd.ApplicationServices.Application, они создаются при первом вызове и далее их можно скрывать и снова делать видимыми, используя свойство Visible набора палеток.

Пример создания палетки на WPF

  1. Создаём UserControl

 

Назовем класс окна палетки PalletteWpfChild. В палетке будет одно переменное текстовое поле NumEntsLabel и одна кнопка UpdateBtn. В переменном текстовом поле будет выводиться количество объектов в активном пространстве документа, а кнопка будет вызывать метод пересчета объектов. 

XAML-код палетки будет таким: 

 

Копировать Код
<UserControl x:Class="PaletteWpfChild" xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation" xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml" xmlns:mc="http://schemas.openxmlformats.org/markup-compatibility/2006" xmlns:d="http://schemas.microsoft.com/expression/blend/2008" mc:Ignorable="d" d:DesignHeight="300" d:DesignWidth="300" Background="White"> <Grid Width="300"> <StackPanel HorizontalAlignment="Left" VerticalAlignment="Top" Margin="8" Orientation="Horizontal" Width="300"> <Label Content="Количество объектов:"/> <Label Name="NumEntsLabel" Content="0" MinWidth="50"/> </StackPanel> <Button Name="UpdateBtn" Content="Обновить" HorizontalAlignment="Left" Height="22" Margin="14,45,0,0" Width="73" VerticalAlignment="Top" Click="UpdateBtn_Click"/> </Grid> </UserControl>

 

Бэкэнд палетки будет таким:

Копировать Код
using System.Windows; using Teigha.DatabaseServices; using HostMgd.ApplicationServices; // Класс, определяющий интерфейс палетки public partial class PaletteWpfChild : System.Windows.Controls.UserControl { // В конструкторе задаем начальную ширину палетки public PaletteWpfChild() { InitializeComponent(); this.Width = 300; } // Отработка события нажатия кнопки private void UpdateBtn_Click(object sender, RoutedEventArgs e) { NumEntsLabel.Content = CalculateObjects().ToString(); } // Метод, рассчитывающий количество объектов private int CalculateObjects() { // Получаем доступ к чертежу, далее к его базе данных, чтобы посчитать количество объектов Document doc = Platform.ApplicationServices.Application.DocumentManager.MdiActiveDocument; Database db = doc.Database; // Инициализируем переменную для рассчета int curSpaceObjects = 0; // Начинаем транзакцию с базой данных чертежа using (Transaction trans = db.TransactionManager.StartTransaction()) { // Получаем доступ к активному пространству документа BlockTableRecord currentSpace = trans.GetObject(db.CurrentSpaceId, OpenMode.ForRead) as BlockTableRecord; // Перебираем циклом все объекты активного пространства документа и // при каждой итерации изменяем счетчик на +1. foreach (ObjectId obj in currentSpace) { curSpaceObjects++; } } // Возвращаем результат рассчета return curSpaceObjects; } }

 

  1. Создаем команду, которая будет добавлять палетку к интерфейсу:

 

Копировать Код
using System; using Teigha.Runtime; // Класс, где будет расположен метод-команда public partial class Commands { // Создаем статические переменные для набора палеток и палетки static HostMgd.Windows.PaletteSet paletteSet; static HostMgd.Windows.Palette paletteWpf; Определяем метод-команду [CommandMethod("HelloWPFPalette")] public void HelloWPFPalette() { // Проверяем, есть ли уже в приложении набор палеток if (paletteSet == null) { // Если набора нет, то создаём новый, задаём ему GUID и название paletteSet = new HostMgd.Windows.PaletteSet("HelloDotNET PaletteSet", new Guid("6AA64CDC-7CBA-49C5-8A14-0B8775BAC5BC")); // Минимальный и исходный размер палетки paletteSet.MinimumSize = new System.Drawing.Size(300, 300); paletteSet.Size = new System.Drawing.Size(300, 300); } else { // Если этот набор палеток уже существует в приложении, то просто делаем его видимым paletteSet.Visible = true; } // Теперь проверяем, существует ли в приложении наша палетка if (paletteWpf == null) { // Если нет, то создаем экземпляр. Если есть, то она появится при изменении видимости набора палеток paletteWpf = paletteSet.AddVisual("WpfPalette", new PaletteWpfChild()); paletteSet.Visible = true; } // Соответственно, чтобы скрыть палетку, нужно изменить настройки видимости набора палеток paletteSet.Visible = false; } }
